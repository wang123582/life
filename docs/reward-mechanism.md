# 任务与奖励机制优化说明

## 核心原则

奖励不是为了让用户刷任务，而是为了让用户更愿意回到自己设立的秩序中。

因此第一版采用的是：

- 完成一个有效闭环
- 解锁一个有上限的放松窗口

而不是：

- 做一个任务
- 换一次无限娱乐

## 什么叫“有效闭环”

第一版中，以下行为都可以触发放松窗口：

1. 完成一个有效番茄钟
2. 完成一个今日任务
3. 完成一个固定生活任务

这意味着产品奖励的不是“勾数量”，而是：

- 开始了
- 推进了
- 没有继续空转

## 为什么不能直接做成“做任务换娱乐”

如果奖励太直接，会出现几个问题：

1. 用户倾向于拆很多水任务
2. 用户为了娱乐而不是为了目标推进去做任务
3. 难度高但真正重要的任务反而吃亏
4. 产品关系容易退化成“自我交易系统”

## 第一版采用的边界

### 1. 奖励有上限

放松窗口有分钟数限制，不无限续杯。

### 2. 奖励内容偏低刺激

第一版推荐的是：

- 影视解说
- 轻松视频
- 听点东西
- 出去走走

不鼓励直接跳进高刺激内容黑洞。

### 3. 奖励在完成后给，而不是提前借支

先完成一个闭环，再放松。

### 4. 奖励窗口会过期

放松窗口不是永久通行证，而是有限时间的缓冲。

## 后续可以继续优化的方向

### 1. 按闭环质量而不是数量奖励

例如：

- 有效番茄钟 > 普通完成勾选
- 高价值任务 > 低价值任务
- 困难后继续推进 > 无难度任务完成

### 2. 奖励类型分级

- 短休息：5~10 分钟
- 中休息：15~20 分钟
- 完整放松：30 分钟

### 3. 奖励内容个性化

后面可以根据用户偏好推荐：

- 看什么
- 听什么
- 走动提醒
- 社交提醒

### 4. 把“交流”也纳入奖励逻辑

例如：

- 完成一轮后，不只是推荐刷内容，也可以推荐联系一个人
- 完成一天后，推荐一次现实中的轻交流

## 当前判断

第一版最稳的奖励机制不是“多做多奖”，而是：

> 做完一个有效闭环，就允许自己有边界地放松一下。

这会更接近正常生活，也更符合 `life` 这个产品本来的方向。
