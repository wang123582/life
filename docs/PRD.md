# life 第一版 PRD

## 产品目标

`life` 的目标不是让用户做更多事，而是帮助用户把一天过得正常、清楚，不再在空转、刷手机和拖延中虚度光阴。

第一版服务对象只有一个人：当前使用者自己。

## 目标用户

- 想维持正常生活节奏的普通人
- 不想每天都觉得自己什么都没做
- 容易在任务太大、手机分心、状态不稳时滑走
- 愿意接受提醒、番茄钟和一定程度的干预

## 核心问题

1. 每天打开一个空白待办页，不知道该写什么
2. 任务太大，难以开始
3. 刚开始做就被手机或其他刺激带走
4. 卡住时没有机制继续推进
5. 工作之外的吃饭、休息、洗澡、交流等生活任务经常被忽略
6. 复盘时只会责怪自己，不知道自己最常掉进什么状态

## 第一版产品原则

1. 先有任务池，再决定今天做什么
2. 每个任务都要能拆到最小完成单元
3. 卡住不是结束，而是生成下一步的入口
4. 生活任务和主动任务同等重要
5. 奖励的是“回到秩序”，不是“刷勾数量”
6. 产品语言日常、明确、不过度说教

## MVP 范围

### P0

- 任务池
- 今日任务列表
- 固定生活任务
- 最小任务拆解
- 番茄钟
- 困难记录与“转成下一步”
- 状态记录
- 今日不做清单
- 交流提醒
- 放松窗口解锁
- 日复盘
- 日 / 周模板

### P1

- 更完整的周复盘
- 更细的提醒配置
- 更强的站点/应用阻断
- 可编辑的模板库
- 跨设备同步

### 明确不做

- 社区
- 排行榜
- 多人协作
- AI 自动拆任务
- 复杂积分系统
- 项目管理式高级字段

## 用户主流程

1. 在任务池中整理长期任务与固定生活任务
2. 从任务池挑选任务放入今天
3. 为今日任务拆出一个最小动作
4. 开始一个番茄钟
5. 番茄结束后记录结果：完成 / 未完成 / 被卡住
6. 如果卡住，记录困难并生成下一个最小动作
7. 完成一个有效闭环后，解锁一个有限时长的放松窗口
8. 晚上做极简复盘，准备明天第一步

- 今日概况
- 今天要做

### 2. 任务池页

- 固定生活任务

### 3. 模板页

### 4. 复盘页

- 周模板预览

## 关键交互

### 任务池 → 今天

- 任务池条目直接支持“放进今天”
- 不要求用户重复录入复杂字段

### 任务 → 最小下一步

- 每个任务都能追加一个最小动作
- 示例文案引导用户写出可立即开始的动作

### 番茄结束 → 困难转下一步

- 系统在番茄结束后要求记录结果
- 如果未完成，要求记录困难类型与下一步动作
- 一键把下一步动作追加回原任务

### 放松窗口

- 完成有效闭环后解锁
- 有时长上限
- 推荐低刺激放松内容

## 提醒与干预

### 轻提醒

- 文案鼓励
- 今日目标提示
- 番茄结束提示

### 软阻断

- 专注时段展示重点黑名单目标
- 以可见提醒和二次确认作为第一版软阻断

### 生活提醒

- 吃饭
- 休息
- 洗澡
- 洗衣服
- 与人交流

## 指标（本地体验版）

第一版先不做埋点系统，但可用界面数据观察以下结果：

- 今日任务完成数
- 有效番茄钟数量
- 困难记录数量
- 状态记录数量
- 是否完成一次真实交流
- 是否出现放松窗口解锁

## 一句话验收标准

用户能在 3 分钟内完成“从任务池选任务 → 拆出最小动作 → 开一个番茄钟 → 记录困难并生成下一步 → 获得放松窗口”的闭环，这一版就算成立。
